## 正文（writer 初稿，经 reviewer 评审）

Agent 系统的核心能力在于通过大语言模型（LLM）与外部世界的交互来完成复杂任务。从最初的基础工具调用，到引入循环机制、检索增强（RAG）、记忆系统，再到最终采用图结构进行控制流编排，Agent 架构经历了一系列深刻的演进。本文将围绕这五个核心维度，系统梳理 Agent 关键机制的实现原理与设计思想。

### 1. Tool Calling 基础实现原理与 API 差异
Tool Calling 的核心流程跨 LLM 通用，均遵循“定义工具 → 模型决策 → 执行工具 → 返回结果 → 模型综合”的链路 [第二周学习复盘#两者的本质是相同的]。其关键设计原则在于模型本身不执行工具，仅负责决策调用目标，而实际的执行权、参数校验与权限拦截均掌握在开发者代码手中 [第二周学习复盘#认知2]。在架构实现上，典型 Agent 的核心骨架通常由一个 while 循环与工具注册表构成，通过判断模型的 `finish_reason` 来决定是正常退出还是继续执行工具 [第二周学习复盘#Agent Loop 核心模式]。此外，各厂商在 API 层面存在关键差异，例如 Anthropic 采用扁平结构的 `input_schema`，而 OpenAI 和千问等厂商则采用标准的 JSON Schema 结构来定义工具参数 [第二周学习复盘#关键差异点]。

### 2. 从单次调用到 Agent Loop 的演进机制
单次工具调用仅能处理“查一次即结束”的简单场景，无法支撑需要多步操作与多轮决策的真实复杂任务 [第二周学习复盘#Agent Loop 核心模式]。早期实测暴露出单次调用存在退出条件过于简单、缺乏 fallback 机制以及上下文无限增长等局限 [第二周学习复盘#当前实现的不足（本周发现）]。为此，Agent Loop 应运而生，其本质是将人类“接到任务-思考-查询-判断-再查询-输出”的工作方式代码化 [Agent_Loop_设计笔记#1.1 最简模型]。通过构建“思考-行动-观察”的循环机制，系统能够反复询问模型下一步动作，直到模型确认任务完成，从而有效处理多步推理需求 [Agent_Loop_设计笔记#1.1 最简模型]。

### 3. 从静态 RAG 到 Agentic RAG 的范式演变
随着长上下文窗口的普及，传统的“切块-入库-拼接”一次性朴素 RAG 流水线在小语料场景中逐渐退场 [第四周第五周学习复盘#认知1]。当前主流形态已演化为 Agentic RAG，即 RAG 从产品决策下沉为基础设施，成为 Agent 按需调用的检索工具 [第四周第五周学习复盘#认知1]。两者的核心区别体现在两个方面：一是推理路径的归属，由人写死的静态管道转变为模型自主决定何时检索；二是是否具备 maker-checker 循环，从一次性的 retrieve-then-read 升级为“评估-精炼-可能再检索”的迭代循环 [第四周第五周学习复盘#周二 — 设计方案 + RAG 概念厘清]。

### 4. 记忆系统的分层设计与实现机制
缺乏记忆会导致 Agent 每轮对话独立，无法理解上下文指代，而 RAG 与短期聊天历史并不能等同于真正的长期记忆 [第四周第五周学习复盘#周一 — 概念输入与认知建立]。记忆系统的设计难点不在于存储，而在于每轮调用前如何精准挑选进入 prompt 的内容 [第四周第五周学习复盘#认知 3]。在架构上，记忆系统通常分为两层：短期记忆局限于会话级（thread 内），主要包含对话历史与工作区状态；长期记忆则跨越用户级（跨 thread），用于持久化用户偏好与核心事实 [第四周第五周学习复盘#周一 — 概念输入与认知建立]。

### 5. 基于 LangGraph 的图结构控制流管理
随着 Agent 逻辑复杂度的提升，手写 while 循环面临严峻挑战：主循环代码极度膨胀，状态管理完全依赖 messages 列表导致无法持久化，且中途出错时缺乏干净的重试与恢复机制 [第三周学习复盘#六、对后续 Roadmap 的思考]。为突破这一天花板，LangGraph 引入了“图（Graph）而非链（Chain）”的核心洞察 [第六周学习复盘#二、本周做的事，一句话]。通过构建有向循环图，系统能够将复杂的 Agent 逻辑切分为独立的节点，实现精细化的状态管理与控制流编排，从而有效解决失败路径无法推理的问题 [第六周设计草稿#五、状态图（定稿）]。

### 结论
Agent 系统的演进是一场从线性执行到网状编排、从无状态交互到具备长期记忆的认知升级。通过夯实 Tool Calling 基础、引入 Agent Loop 与 Agentic RAG、构建分层记忆系统，并最终采用 LangGraph 等图结构进行控制流管理，开发者能够打造出更加健壮、智能且具备高度容错能力的复杂 AI 应用。

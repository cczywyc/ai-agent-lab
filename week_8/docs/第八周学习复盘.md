## 二、本周做的事，一句话

把第七周 v5.0 那个"一张 LangGraph 图、planner/executor/critic 共享 state"的角色**节点**，升格成 **supervisor 协调的独立 workers**：supervisor 拆研究子任务、researcher 逐个检索压缩 findings、writer 把 findings 组织成初稿、reviewer 二元 rubric 审稿出 verdict、reject 则带 review_notes 返修（最多 `MAX_REVIEW=2`，达上限取 best-so-far）。**核心认识：本周是把角色节点升格成独立 workers，不是重写引擎**——researcher = executor 引擎升格、reviewer = critic 升格、supervisor = planner 升格，真正"新"的只有 writer 角色、工具式 handoff（task_description 四要素）、writer↔reviewer 打回循环三块。

**这周和第七周的关系，是"降一层复用"的又一次成立**——而且这次有了 token 账的硬证据。第七周是把第六周单层那套**整体降一层**；第八周是把第七周"一张图里的角色"**升格成独立 worker**，复用的远多于新增的：

| 第七周机制（一张图） | 第八周升格 |
|---|---|
| planner（拆计划、判 done） | supervisor（拆研究子任务、收编 findings、按阶段派活） |
| executor 引擎（agent↔tools↔inject_\*） | researcher（**内层引擎原样复用**，只换了入口投影 + 出口收编） |
| critic（单步自检 accept/retry/escalate） | reviewer（draft 级审稿 accept/reject + score + notes，critic 升格） |
| finalize（代码装配 step_results） | writer（**全新** LLM 写作角色，跨 findings 综合成稿） |
| escalate / skip-and-advance / `replan_count` | writer↔reviewer 打回循环 / `review_count`（第七周断路器思路降一层） |
| §四分层装配（共享池里按角色挑段） | 隔离投影（每 worker 一个显式投影函数，喂 LLM 的是投影不是 state） |

**升格不重写有多硬，看 token 账（§五）**：researcher（继承的 RAG 引擎）独占 token 大头（A/B/C **79–85%**、D 88.8%），v6.0 新增的 writer+reviewer 打回层只是小额加价（A/B/C **8–12%**、约 24–26k 每题；D 打回只 1 轮约 3.8%）。新增的少、复用的多——这就是"blast radius 最小"的量化证据，不是口号。

第七周复盘 §七留了一个问题进第八周，《多 Agent 概念》§十一 又列了 Q1–Q4 带进设计日。逐条兑现，且**多数由真实跑给了实证回应**：

| 上周 / 周一说的 | 本周结果 |

| Q3：跨 agent 控制流，纠正/降级在哪一层？（承第三周 Q3，跨两周悬念） | ✅ **结案**：内层（researcher 内）`turn_count`+synthesis-reserve+空回答重试；外层（supervisor 级）`review_count`+`replan_count`+recursion guard。Topic D 真实跑坐实：QSCC 子任务内层失败 → 压缩回传 → supervisor skip-and-advance（replan+1），**不穿透成全局崩溃**。 |
| Q4：委派契约第 4 条 boundary 在 3 个固定异质角色时还重要吗？ | ◑ **部分兑现 + 一处反例**：桩 E2 验 boundary 原样传到 worker；真实跑里 3 异质角色顺序执行天然不撞车、boundary 弱化**符合预判**。但 §六 #5 暴露**另一种"漂"**——supervisor 把"本项目"题拆成通用子任务，researcher web 漂移、还幻觉出 `edit` 工具名。所以"边界"在本周不是"别碰别人的活"，而是"别丢本地库语境锚"——边界的形态变了、没消失。 |

四条设计问题全有了答案，且 Q1/真·多 agent 之问是**真实跑数据**给的实证回应——这正是周一"带着怀疑去学、别一上来堆 agent"那根刺的兑现：拿自己的 demo 称了称，确认这条顺序流水线本就在多 Agent 的非甜区。

## 五、实验与量化

### 桩节点验"框架接线"（E1–E7，7/7，每条配对照组）

零 API、确定、可复现，只验框架机制不验业务质量。E1 supervisor 路由正确性 / E2 task_description 四要素 / E3 review_count 闸门收口 / E4 两计数器不串扰 / E5 上下文隔离 / E6 best-so-far 收口 / E7 双层闸门嵌套。结论分三类（承第七周）：**巩固型**（E2/E3/E4/E6 升格后照旧成立）+ **发现型**（E1 计划被证伪、E5 文档缺口）+ **scope 诚实**（E7 一条 sub-claim 属继承）。

- **E5 是本周唯一真·新机制**（A→B 没有 v5.0 对应物）：LangGraph **不分区**——每个 worker 物理上收到全 state dict（对照组实测 writer 越界读到 17 键、含 reviewer 私有 `draft_score/review_verdict/best_draft/review_notes`），越界读**不报错、只静默串台**、框架无护栏。所以隔离 = **每 worker 一个显式投影函数**，喂 LLM 的只能是投影结果、不能是 `state` 本身。
- **E1 计划被证伪**：原计划拿 prebuilt `create_supervisor` 当桩测对拍基线——实测它是 LLM 调 handoff 工具的决策、**非零 API**，破"桩 worker 不接 LLM"第一纪律。改用零 API 的"写歪路由"探针（routing_accuracy 命中 1/3）。

### 离线桩测全绿（实现期，零 API）

| 套件 | 判据 | 验什么 |
|---|---|---|
| `test_graph.py` | **48/48**（M1–M9） | E1–E7 各机制搬进真实 v6.0 图端到端复跑 + 隔离投影 + 打回循环 |
| `test_criteria.py` | **23/23** | 占位符识别 / 判据纯函数（v4.2 单步口径保真） |
| `test_store_memory.py` | **56/56** | Store×记忆 e2e（外循环适配 + 引用白名单兜底） |

**离线合计 127/127 + 桩测 E1–E7 7/7 全过。**

### 真实 API 端到端（4 题，2026-06-15）

| 题 | 压的路径 | 子任务 | review 轨迹 | 收口 | 隔离 | model |
|---|---|---|---|---|---|---|
| A | 打回循环+Reflexion | 4 ok / 0 skip | reject 0.85 → accept 0.95 | accept 早退 | clean | max |
| B | skip-and-advance | 3 ok / 0 skip | reject 0.40 → accept 0.95 | accept 早退 | clean | max |
| C | 5 子任务长链纵向对比 | 5 ok / 0 skip | reject 0.65 → accept 0.95 | accept 早退 | clean | max |
| D | 强制 skip（杜撰 QSCC） | 1 ok / **2 skip** | accept 0.95 | accept 早退 | clean | plus |

- **打回循环主力路径坐实**：A/B/C 初稿都被 reject、返修后 score **全升**到 0.95；reviewer 落"健康中段"（不太松——3 题初稿都 reject；不太严——从不连续 reject 两次）。
- **skip-and-advance + 正交计数器 + 防御收口坐实（D）**：杜撰 QSCC 模块逼出 `replan_count=2 / review_count=0`（两计数器真实跑互不吃额度），2/3 子任务 skip 后 ok_findings=1 时 writer **仍写出成稿**。
- **隔离投影真模型复证**：4/4 题 isolation_clean，三 worker 真实可见集精确 = 设计视图、**没串台**。
- **内层接地软闸门（继承 v5.0）真实有效**：B step0「49 cites grounding=0.02 → retry」、step2「35 cites 0.46 → retry」、C step1 retry——真模型下抓出引用堆砌/编造，retry 后 accept。
- **成本账**：researcher 独占 79–89% token（A/B/C 79–85%、D 88.8%）、新增 writer+reviewer 仅 3.8–12.1%；4 题合计 **~103 万 token 一个 session 烧穿 qwen3.7-max 免费额度**（A/B/C 用 max、跑完即 403；D 改用 qwen3.7-plus 残量）。

## 六、踩坑与本周发现

### 最该记的一条：离线验接线、真实验策略——又深了一层

第七周教训是"离线 64/64 全绿、真实跑连摔三跤"。第八周更精确地认识到**桩测和真实跑验的根本不是同一件事**：

> **桩测验的是断路器路径（E3 恒 reject 撞闸门收口），真实跑验的是健康中段主力路径（reject→拿 notes 改好→早退）。** 这两者验的是两套行为——断路器只在"一直失败"时触发，主力路径才是正常运行。离线 **127/127** 全绿照样挖出 **2 修 + 3 记**。**离线全绿仍是必要非充分。**

### 5 个坑（已修 2 + 记 3）

**已修（真实跑坐实的代码缺陷，改完离线复验 + D 集成复跑确认）**：

1. **findings 600 截断饿瘦 writer**。A/B/C 共 **12/12 条 ok finding 全触顶 600**——这是隔离的代价：writer 只看投影、看不到研究全轨迹，600 砍太狠就丢信息（C 成稿偏教科书通用、缺版本号级项目细节）。修 `600→1500`，D 复跑实测 2.5× 信息量、仍是压缩摘要不破 §四纪律。
2. **`doc#section` 字面占位泄漏**。SYSTEM/WRITER_PROMPT 的格式范例 `[doc#section]` 被模型原样抄进正文引用汇总。**妙的是这条是 reviewer 自己在 B round-1 抓出来的**（"出现 `[doc#section]` 及 `...#…` 占位符"），不是我 grep 出来的——多 Agent reviewer 兜底 writer 过度的活实证。修 `_extract_citations` 剔字面占位。

**记 v0.3（策略缺口，额度耗尽未复跑、不做未验证改动）**：

3. **skip 后 writer 静默吞掉跳过的子任务、不在正文标注缺口**。Topic D 把 QSCC 整个跳过了，reviewer 还 accept——因为 rubric"扣题"只查"覆盖 findings"、不查"覆盖原始主题"。用户要"LangGraph vs QSCC 对比"，拿到的是只字未提 QSCC 的单边稿，缺口只记在 meta 脚注、不在正文。
4. **引用堆砌**（writer 单句挂 10+ 引用）。B/C round-1 reviewer 都点名。**但打回循环已自愈**——round-2 writer 改掉、score 升到 0.95。这是 §一价值点"角色专精 + 质量门"在真实跑里自己证明了自己（缺陷被多 Agent 结构兜住）。prompt 硬约束待下次真实跑验。
5. **supervisor 把"本项目"题拆成通用子任务 → researcher web 漂移**。C 的子任务被拆成"Tool Calling 的基础实现原理是什么"（丢"我们项目"语境）→ researcher 上 wikipedia/brave 查通用概念、还**幻觉出不存在的 `edit` 工具名**（被 UnknownTool 兜住）→ 靠 synthesis-reserve 才收回本地引用。拆解时丢了"本地库"语境锚。

### 方法论沉淀（这周新长出来的）

- **想压的路径压不到，就造探针逼出来。** 健康中段的 reviewer 从不连续 reject 两次，断路器路径（best-so-far 双 reject、A/B/C 的 skip 收口）压不到——于是造 **Topic D（纯杜撰 QSCC 模块）** 把 skip 稳定逼出来。这和第七周造 E6/E7 补充探针同构：真实语料太"友好"压不出失败时，得人为制造靶子。
- **处置纪律：只修真实跑坐实的，不做额度耗尽下无法复跑验证的改动。** #1/#2 改完离线复验 + D 集成复跑才算数；#3/#4/#5 记下来等下次真实跑压。这是"系统化调试、不靠猜"的延续。
- **scope 诚实贯穿全程。** best-so-far 双 reject 收口真实跑**未触发**、仍只桩 E6 验（诚实记一笔）；D 跑在旧模型且 round-1 即 accept，"模型更松 vs 题型更好写"两变量**没解耦**、不据此结论"旧模型更松"；**skip 难触发本身是个发现**——真实语料有相邻内容会把子任务救活（Topic B 的 week_8 子任务被相邻 week_7 前瞻内容救活、researcher 诚实改写成"规划而非已落地"），researcher 比桩里机敏得多。这条承第七周 E7"继承非复证"一脉。

## 七、对第九周的思考（MCP）+ 带一个问题进去

第九周接 MCP（host/client/server、tools/resources/prompts 统一暴露、写个 MCP server、让 Agent 通过 MCP 调它），产出《MCP 适合接哪些企业系统》。而 v6.0 里 researcher 那套 **手写的工具注册表 + 错误处理**（`retrieve_documents/web_search/fetch_webpage` 硬编码、`UnknownTool` 兜底幻觉工具名）正好是 MCP 要标准化的东西。

**带这个问题进第九周**：

> **MCP 标准化的是"工具怎么暴露 / 发现 / 调用"（接口层），但缺陷 #5 暴露的"researcher 调用漂移、幻觉工具名、结果不接地要 retry、语境锚怎么不丢"这些控制流，MCP 管不管？**
>
> 大概率**不管**——MCP 给的是 typed 工具契约 + 发现/调用协议，"调用失败怎么降级、引用不接地怎么判 retry、子任务怎么 skip-and-advance、语境锚怎么不丢"仍得自己在 loop 里兜。这是反复撞见的同一个主题——**框架给原语、控制流自己兜**（第六周 LangGraph 给图原语、第七周/第八周的闸门和收口全是自己搭）——的下一站。

**顺带记一笔（喂第十一周工具白名单）**：#5 那个"幻觉出不存在的 `edit` 工具名被 `UnknownTool` 运行时兜住"，正是 MCP 的 typed 工具契约要解决的——看 MCP 能不能把这个兜底**从"运行时撞错"提前到"发现期就没有这个工具"**（schema 层拒绝，而非执行层报错）。这条接第十一周的工具白名单。

## 八、跨模型 / 通用设计原则（本周新增，每条挂证据）

- **升格不重写、blast radius 最小，可被 token 账量化**（researcher 占 79–89%、新增打回层仅 3.8–12.1%）：复用多、新增少不是感觉，是成本结构能称出来的。
- **多 Agent 的价值未必是"更强/更快"，可能是"角色专精 + 一道质量门"**（v5.0 20,905 字无门 → v6.0 2,037 字经 rubric 0.95）：顺序流水线买不到真并行，但买到了成稿质量与质量闸门。选拓扑要先认清买的是哪样。
- **隔离不是框架给的、是约定**（E5 + 真实跑 4/4 clean）：LangGraph 不分区、越界多读静默串台无护栏，隔离必须落成每 worker 一个显式投影函数——喂 LLM 的只能是投影、不能是 state。
- **桩测验断路器、真实跑验主力路径，两者验的不是同一件事**（E3 恒 reject vs 真实 reject→改→早退）：离线 127/127 仍必要非充分，真实跑才验"对模型行为的假设对不对"。
- **多 Agent reviewer 能兜底 writer 过度 = "角色专精"的活实证**（#2 占位符、#4 引用堆砌都被 reviewer 抓出 / 打回自愈）：拆出独立 reviewer 的收益，在真实跑里自己证明了自己。
- **隔离与"喂够下游"会打架，旋钮要实测拧**（#1 findings 600→1500）：投影喂得太省会饿瘦下游、喂得太多就破隔离，平衡点靠真实跑量出来（12/12 触顶是硬信号）。
- **想压的失败路径压不到，就造探针逼出来**（Topic D 杜撰 QSCC）：真实语料太友好时人为制造靶子，承第七周 E6/E7 补充探针。
- **复盘/报告数字要在定稿当天能跑出来**（沿用前两周）：本周量化全部来自真实跑当天输出（127/127、7/7、4 题 token 账、20,905↔2,037、reject 0.85/0.40/0.65→0.95、D replan2/review0）。

## 九、本周最大收获

**从"一张图里的角色节点"跨到"supervisor 协调的独立 workers"，工程上不是"加 agent"，而是把第七周的角色逐个升格、再补三块真正新的东西（writer / handoff 契约 / 打回循环）——而"升格不重写"这次第一次被 token 账量化坐实：researcher 独占 79–89%，新增打回层只占个位数到一成出头（3.8–12.1%）。** 能这么顺地升格，是因为第七周已经把"节点干活、边做决策""降一层复用""桩节点隔离验框架""离线验接线、真实验策略"这套吃透了。

但比"升格跑通"更深的两课：

1. **拿自己的 demo 把判据称出来，比转述判据值钱得多。** 周一带着"真·多 agent 多买到什么"的怀疑去学，周五真实跑用 token 账（成本没炸开）+ v5.0↔v6.0 对比（20,905→2,037、有质量门）回答了它——**这条顺序流水线买到的不是真并行，是角色专精 + 质量闸门**。所以《什么时候该用多 Agent》不是抄 Anthropic 的观点，是拿自己 demo 当反例论证出来的。

2. **隔离这把双刃剑，真实跑才看清它的代价侧。** 桩 E5 验"隔离机制成立"（worker 只读投影），真实跑验"隔离会饿瘦下游"（findings 600 截断 12/12 触顶）——隔离让 worker 干净，也让 writer 看不到全轨迹。**"隔离对了"要拆成两句：机制我用桩测过了（投影不串台），代价我用真实跑量了（喂够下游的旋钮要拧）。** 少了后半句，干净的隔离会悄悄饿瘦成稿。

# 第三周周三实验结论

## 实验一：json_object 模式

| 测试 | 结果 | 说明 |
|------|------|------|
| 有结构描述的 system prompt | ✅ | 输出完全按描述的结构返回，keys = [answer, confidence, sources] |
| 最简描述（只说"返回 JSON"） | ✅ | JSON 合法，但结构是模型自己决定的，keys 不可预测 |
| system prompt 完全不提 JSON | ❌ 400 报错 | **千问强制要求 messages 中包含 "json" 关键词** |
| 中文 + 嵌套结构 | ✅ | 嵌套结构正常，中文编码无问题 |

**结论：**
- `json_object` 模式**可用但不可控** — 能保证输出是合法 JSON，但无法约束具体结构
- **硬性限制**：messages 中必须包含 "json" 一词，否则 400 报错
- 想要稳定的字段结构，必须在 system prompt 中明确描述，但即使描述了也无法 100% 保证遵循
- 适合"只要是 JSON 就行"的场景，不适合需要严格 schema 的场景

---

## 实验二：json_schema 模式（strict: True）

| 测试 | 结果 | 说明 |
|------|------|------|
| 基础 schema (answer/sources/confidence) | ✅ | 所有 required 字段齐全，enum 值合法 |
| 中文内容 | ✅ | 中文在 JSON 中编码完全正常 |
| 嵌套 schema (points 数组 + importance enum) | ✅ | 嵌套对象的 enum 也被严格约束，无多余字段 |
| enum 压力测试（诱导非法值） | ✅ | confidence 仍然返回 "high"，没有被诱导成 "very high" |

**结论：**
- `json_schema` + `strict: True` **非常可靠**
- enum 约束严格生效，即使 prompt 试图诱导也不会违反
- required 字段 100% 存在，additionalProperties: false 也被遵守
- 中文内容无任何编码问题
- **这是周四实现 structured output 的首选方案**

## 实验三：tools + response_format 互斥性（最关键）

| 测试 | tools | response_format | 结果 | 行为 |
|------|-------|----------------|------|------|
| A. 基线 | ✅ | 无 | ✅ | 正常调用工具 |
| B. 基线 | 无 | json_object | ❌ 400 | messages 没有 "json" 关键词 |
| C. 两者 | ✅ | json_object | ❌ 400 | 同样的 "json" 关键词限制 |
| D. 两者 | ✅ | json_object | ❌ 400 | 同上 |
| E. 两者 | ✅ | json_schema | ✅ | **json_schema 吃掉了 tools！模型不调用工具，直接返回 JSON** |
| F. 完整循环 | ✅ | json_schema | ✅ | 同 E — 模型跳过工具，直接 JSON 回答 |
| G. 完整循环基线 | ✅ | 无 | ✅ | 正常调用工具，最终回答是纯文本 |

**关键发现：**

### tools + json_object → 400 报错（需要 messages 含 "json"）
这是 json_object 模式的固有限制，跟 tools 无关。如果在 system prompt 中加入 "json" 可以绕过，但仍无法控制结构。

### tools + json_schema → 不报错，但 response_format 优先级高于 tools
**这是最重要的发现：当同时传 tools 和 json_schema 时，模型不会调用工具，而是直接输出符合 schema 的 JSON。** tools 的定义虽然被接受（不报错），但完全被忽略。

### 对方案 C 的影响
**方案 C（同时传 tools + response_format）不可行。** 两者互斥 — 不是 API 层面报错的互斥，而是行为层面的互斥：json_schema 会覆盖 tools 的行为，导致模型永远不调用工具。

---

## 周四改代码的技术决策

基于以上实验，推荐的实现策略：

1. **工具调用阶段**：只传 `tools`，不传 `response_format`，让模型正常选择和调用工具
2. **最终回答阶段**：当模型决定给出最终回答时（finish_reason == "stop"），发起**第二次请求**，传入 `response_format=json_schema`，将自由文本回答格式化为结构化 JSON
3. 或者，在 Agent Loop 的最后一步，用 system prompt 约束 + `json_schema` 做一次"格式化调用"

**不要试图在同一个请求中同时使用 tools 和 response_format。**
